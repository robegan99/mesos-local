Kubernetes on Mesos

GopherCon VM

The following applications are pre-installed and start on boot:

  • Kubernetes - container cluster manager
    • Core Components:
      • API Server (port 8888)
      • Controller Manager
      • Scheduler - Mesos/Kubernetes task/pod scheduler
    • Addons:
      • Kube DNS - domain name server
      • Kube UI - web gui
    • Kubectl - command line interface (in PATH as kubectl)
    • Source: ~/kubernetes
  • Mesos - distributed systems kernel
    • Core Components:
      • 1x Master (port 5050)
      • 1x Slave (port 5051)
    • Mesos UI - web gui
  • Etcd - key-value store
    • 1x Node (port 4001)
  • Go - programming language
    • GOPATH=~/src/go
  • Docker - container engine
  • Docker Compose - multi-container application manager
  • jq - json parser

Local User Interfaces

  • Kubernetes UI
  • Mesos UI

Kubernetes Walkthrough

  • Kubernetes 101 - Kubectl CLI & Pods
  • Kubernetes 201 - Labels, Replication Controllers, Services and Health Checking
  • Guestbook Example

Caveats:

  • The kubectl binary is already installed on the Gophercon VM.

  • To make the GopherCon VM Kubernetes Pods & Services accessible from your host machine add the following routes on the host:

    sudo route -n add -net 172.17.0.0 10.141.141.10
    sudo route -n add -net 10.10.10.0 10.141.141.10

    To delete them later:

    sudo route delete 172.17.0.0 sudo route delete 10.10.10.0

Bleeding Edge

By default, the GopherCon VM starts up a local Kubernetes from pre-compiled binaries.
To use the bleeding edge, or make modifications to Kubernetes itself, you can build locally and deploy locally with docker.
In order to do so, the local Etcd, Mesos, and Kubernetes need to be stopped first:

sudo service kube-controller-manager stop
sudo service kube-scheduler stop
sudo service kube-apiserver stop
sudo service mesos-slave stop
sudo service mesos-master stop
sudo service etcd stop

To build and deploy Kubernetes, Mesos, and Etcd as docker images, follow the Kubernetes-Mesos on Docker - Getting Started Guide.

Caveats:

  • Use the Mesosphere Kubernetes GopherCon branch checked out at ~/kubernetes (instead of GoogleCloudPlatform's master)
