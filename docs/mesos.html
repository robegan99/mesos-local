Apache Mesos 101

General concepts

Components

Master
One of a quorum of processes responsible for resource aggregation and allocation. Makes resource offers to schedulers. Only one master is active at a time.
Allocator
Pluggable part of the master responsible for forming resource offers. The default allocator implements DRF (dominant resource fairness).
Slave
Local agent that runs on every worker node. Responsible for announcing node-local resources, monitoring running processes and brokering communication between executors and the active master.
Framework
Implements application-specific scheduling and execution logic. Consists of two parts: the scheduler and (optionally) a custom executor.
Scheduler
Part of a framework that registers with the master and launches tasks using resource offers it receives. Has fine-grained control over how tasks execute on the slave via the TaskInfo protobuf message.
Executor
Part of a framework that is responsible for launching tasks. Mesos provides a CommandExecutor that knows how to execute /bin/sh -c <your-command>.

Tasks do not have to be full-blown processes! They could be urls, text snippets, signal data, etc.
Containerizer
Pluggable part of the slave that is responsible for setting up the container around the executor’s process hierarchy. Mesos ships with two of these: the Mesos containerizer and the Docker containerizer).
Isolator
Pluggable part of the Mesos containerizer responsible for monitoring resource usage and enforcing that the executor process hierarchy usage does not exceed the resources allocated to it.

For example, the cgroups/mem isolator sets up memory.limit_in_bytes appropriately, relying on the linux kernel to kill executor hierarchies that overflow their allocations.

Two-level scheduling

Mesos delegates application-specific scheduling decisions to sub-schedulers called Frameworks. The Mesos master ensures that each framework gets a fair share of the total cluster’s resouces, potentially modified by the framework role weights.

Resource offers

In order to do work on the cluster, schedulers need to accept one or more resource offers from the master.

Resource offers look like this: 

/**
 * Describes some resources available on a slave. An offer only
 * contains resources from a single slave.
 */
message Offer {
  required OfferID id = 1;
  required FrameworkID framework_id = 2;
  required SlaveID slave_id = 3;
  required string hostname = 4;
  repeated Resource resources = 5;
  repeated Attribute attributes = 7;
  repeated ExecutorID executor_ids = 6;
}

Resources in the offer look like this: 

/**
 * Describes a resource on a machine. A resource can take on one of
 * three types: scalar (double), a list of finite and discrete ranges
 * (e.g., [1-10, 20-30]), or a set of items. A resource is described
 * using the standard protocol buffer "union" trick.
 */
message Resource {
  required string name = 1;
  required Value.Type type = 2;
  optional Value.Scalar scalar = 3;
  optional Value.Ranges ranges = 4;
  optional Value.Set set = 5;
  optional string role = 6 [default = "*"];
}

The scheduler interface

See the Mesos-Go native language bindings on GitHub. 

type ExampleScheduler struct{}

func (sched *ExampleScheduler) Registered(driver sched.SchedulerDriver, frameworkId *mesos.FrameworkID, masterInfo *mesos.MasterInfo) {
}

func (sched *ExampleScheduler) Reregistered(driver sched.SchedulerDriver, masterInfo *mesos.MasterInfo) {
}

func (sched *ExampleScheduler) Disconnected(sched.SchedulerDriver) {}

func (sched *ExampleScheduler) ResourceOffers(driver sched.SchedulerDriver, offers []*mesos.Offer) {
	for _, offer := range offers {
		// launch or decline
	}
}

func (sched *ExampleScheduler) StatusUpdate(driver sched.SchedulerDriver, status *mesos.TaskStatus) {
	log.Infoln("Status update: task", status.TaskId.GetValue(), " is in state ", status.State.Enum().String())
}

func (sched *ExampleScheduler) OfferRescinded(sched.SchedulerDriver, *mesos.OfferID) {
}

func (sched *ExampleScheduler) FrameworkMessage(sched.SchedulerDriver, *mesos.ExecutorID, *mesos.SlaveID, string) {
}

func (sched *ExampleScheduler) SlaveLost(sched.SchedulerDriver, *mesos.SlaveID) {
}

func (sched *ExampleScheduler) ExecutorLost(sched.SchedulerDriver, *mesos.ExecutorID, *mesos.SlaveID, int) {
}

func (sched *ExampleScheduler) Error(driver sched.SchedulerDriver, err string) {
}
